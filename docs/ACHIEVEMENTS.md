# Achievement atlas

This atlas separates current facts from community observations. GitHub officially describes Achievements as profile badges for specific events and actions, but does not publish a complete, stable list of triggers and tier thresholds. The feature is in public preview and can change.

## Read the labels first

| Label | Meaning |
|---|---|
| Official | GitHub documents the profile behavior or setting. |
| Observed | Repeated community evidence exists; treat the trigger or number as a checkpoint, not a contract. |
| Historical | Existing holders may still show it, but the original event has passed. |
| Unverified | Current reproducibility is unclear; do not build automation around it. |

## Visible tier model

Tiered achievements display the base badge and multipliers through **`x4`**. There is no visible `x8` tier. Once `x4` is reached, more qualifying activity may still be meaningful work, but it does not create a higher visible multiplier.

### Community-observed checkpoints

| Achievement | Base | `x2` | `x3` | `x4` | Confidence |
|---|---:|---:|---:|---:|---|
| Pull Shark | 2 | 16 | 128 | 1,024 | widely observed |
| Galaxy Brain | 2 | 8 | 16 | 32 | widely observed |
| Starstruck | 16 | 128 | 512 | 4,096 | widely observed |
| Pair Extraordinaire | 10 | 24 | 48 | 64 | community-observed; verify before scaling |

Counts are included for orientation. They are not a reason to generate empty work. Stop at the smallest useful experiment and wait for propagation before doing anything else.

## Active and reproducible

### Pull Shark

Have pull requests you authored merged. A clean route is normal product work: a focused branch, a reviewable change, and a merge by a maintainer or collaborator. Achievement Lab can reproduce the event only inside its isolated repository.

**Do:** useful PRs, honest review, descriptive history.  
**Don't:** flood repositories with timestamp-only changes to chase a tier.

### Galaxy Brain

Provide answers in GitHub Discussions that the question author marks as accepted. It is tiered through `x4`; `x4` is the visible maximum.

The runner intentionally leaves this manual. An accepted answer should solve a real question, and Discussions setup differs between repositories.

### Pair Extraordinaire

Create commits that GitHub credits to multiple authors using valid `Co-authored-by` trailers. Both authors should have contributed to the change and the email in the trailer must map to the co-author's GitHub account.

```text
Co-authored-by: Ada Example <12345+ada@users.noreply.github.com>
```

### Starstruck

Own a repository that other people choose to star. This is a social signal, not a lab event. The only professional strategy is to build something useful, present it clearly, and let stars remain voluntary.

### Quickdraw

Close an issue shortly after opening it. One controlled issue is enough to test the event; repeating it has no visible tier benefit.

### Public Sponsor

Publicly sponsor open-source work through GitHub Sponsors. This involves money and a real support relationship. It is never automated by this project.

## Experimental or uncertain

### YOLO

The commonly cited trigger is merging a pull request without review. The runner can reproduce that event in a lab repository, but current awarding is inconsistent enough that this guide treats it as an **event test**, not a promised achievement.

### Heart On Your Sleeve and Open Sourcerer

These names appear in community lists, but current triggers and availability are not reliable enough for this project to prescribe or automate. Keep them in the research column until fresh evidence exists.

## Historical achievements

### Arctic Code Vault Contributor

Awarded for contributions included in GitHub's 2020 Arctic Code Vault snapshot. The underlying historical event cannot be recreated today.

### Mars 2020 Helicopter Contributor

Connected to contributions to qualifying repositories used by the Mars 2020 Helicopter mission. It is historical, not a repeatable profile optimization.

## Not achievements

Do not confuse these with GitHub Achievements:

- workflow-status, coverage, version, license, and technology badges in Markdown;
- repository topics such as `python` or `github-api`;
- program badges such as GitHub Developer Program membership;
- contribution graph visuals generated by third-party services.

They can all improve navigation or credibility, but they are different systems and should be labelled honestly.
